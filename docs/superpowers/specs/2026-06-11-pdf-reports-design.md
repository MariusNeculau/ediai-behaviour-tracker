# Real PDF Reports (Individual Child) — Design Spec

**Date:** 2026-06-11
**Status:** Approved (pending spec review)

## Problem

The Reports tab has a "Generate Report" form and a static "Sample Report Preview" with
hardcoded data, but the Generate button only shows a "not yet implemented" toast. Users
cannot produce a real, downloadable report from their logged incident data.

## Goal

Generate a real, downloadable **PDF** for an **Individual Child** over a selected time
period, built from the child's actual incidents. Server-side, pure-Python (ReportLab),
streamed to the browser as a file download — no files stored on disk.

## Non-Goals (v1)

- Class Summary and Whole School reports (fast-follow; the form keeps them but they stay
  on the existing "not yet implemented" toast).
- Persisting generated reports / a populated "Recent Reports" list (download is on-the-fly).
- LLM/AI-written narrative or recommendations (content is deterministic aggregation).
- Calendar-anchored periods (v1 uses rolling windows — see below).

## Periods (rolling windows)

Computed from `today` (the server's current date). A report includes incidents whose
`occurred_at` date is `>= start` and `<= today` (inclusive):

| period | window | start = |
|--------|--------|---------|
| `week` | last 7 days  | `today - 6 days` |
| `month`| last 30 days | `today - 29 days` |
| `term` | last 90 days | `today - 89 days` |

"Last 7 days" means today plus the 6 prior days (7 calendar days inclusive). `month` and
`term` follow the same inclusive convention. Default period is `month`.

## Architecture

Three units with clear boundaries:

### `reports.py` (pure logic — no Flask, no DB session)

- `period_start(period, today)` → `date`
  - Maps `week`/`month`/`term` to the start date per the table above.
  - Raises `ValueError` for an unknown period (the endpoint turns this into a 400).
- `build_child_report(child, incidents, period, today)` → `dict`
  - `child` is a `Child` model instance (read-only attribute access: `name`, `room`,
    `key_worker`); `incidents` is a list of `Incident` instances already filtered to the
    period; `period`/`today` provide label + date range.
  - Returns a plain dict (the report "view model"):
    ```python
    {
        "school": str,            # filled by the endpoint, see below
        "child_name": str,
        "room_name": str,         # child.room.name or "—"
        "key_worker": str,        # child.key_worker.name or "—"
        "period_label": str,      # "Last 30 days"
        "period_range": str,      # "13 May 2026 – 11 Jun 2026"
        "generated_on": str,      # "11 Jun 2026"
        "total_incidents": int,
        "per_week_avg": float,    # round(total / (window_days / 7), 1)
        "top_severity": str,      # mode of severity, or "—" if none
        "avg_duration": str,      # f"{round(mean)} min" over incidents with duration, else "N/A"
        "incident_rows": list,    # [{"date","type","severity","trigger","outcome"}], desc by occurred_at
        "top_trigger": str,       # mode of non-empty triggers, or "—"
        "top_type": str,          # mode of type, or "—"
        "peak_time": str,         # "Morning (before 12:00)" | "Afternoon (12:00–17:00)" | "Evening (after 17:00)" | "—"
        "pattern_text": str,      # deterministic sentence built from the above, or empty-state text
    }
    ```
  - `window_days` per period: week=7, month=30, term=90.
  - `per_week_avg = round(total_incidents / (window_days / 7.0), 1)`.
  - `top_severity` / `top_trigger` / `top_type`: most frequent value; ties broken by first
    occurrence in the sorted (desc) list; empty inputs → `"—"`.
  - `avg_duration`: mean of `incident.duration` over incidents where `duration` is not None,
    rounded to nearest int, as `"{n} min"`; if none have a duration → `"N/A"`.
  - `peak_time`: bucket each incident's `occurred_at.hour` — Morning (`hour < 12`),
    Afternoon (`12 <= hour < 17`), Evening (`hour >= 17`); report the most frequent bucket
    as `"Morning (before 12:00)"` / `"Afternoon (12:00–17:00)"` / `"Evening (after 17:00)"`;
    no incidents → `"—"`. Ties broken by Morning < Afternoon < Evening order.
  - `pattern_text`: if `total_incidents == 0` → `"No incidents recorded in this period."`
    Otherwise a deterministic sentence, e.g.
    `"Most incidents were {top_type} type, most often triggered by {top_trigger}, "`
    `"with a peak in the {peak_time_short}."` (Omit clauses whose value is `"—"`.)
- `render_child_report_pdf(report)` → `bytes`
  - Builds the PDF with ReportLab Platypus into an in-memory `BytesIO` and returns
    `buffer.getvalue()`.
  - Layout mirrors the existing "Sample Report Preview" sections: header band (title +
    school + generated date), meta line, stats row, Incident Summary table, Pattern
    Analysis paragraph, footer ("Generated by EDI AI · {school}" + "CONFIDENTIAL").
  - Empty incident list → the table is replaced by a single "No incidents recorded in this
    period." line; stats show 0 / 0.0 / — / N/A.

### `reports_api.py` (Flask blueprint, url_prefix `/api`)

- `GET /api/reports/child/<int:child_id>`
  - Query param `period` (default `"month"`).
  - `child = db.session.get(Child, child_id)`; if `None` → `404 {"error": "Unknown child"}`.
  - `try: start = period_start(period, date.today())` `except ValueError: → 400 {"error": "Invalid period"}`.
  - Query incidents: `Incident.query.filter(Incident.child_id == child_id,
    Incident.occurred_at >= datetime(start), Incident.occurred_at < datetime(today + 1 day))
    .order_by(Incident.occurred_at.desc()).all()`.
  - `report = build_child_report(child, incidents, period, date.today())`.
  - `report["school"] = serialize_system_config(SystemConfig.query.first())["name"]`.
  - `pdf = render_child_report_pdf(report)`.
  - Return `Response(pdf, mimetype="application/pdf", headers={"Content-Disposition":
    f'attachment; filename="{filename}"'})`.
  - `filename = f"EDI_AI_Report_{safe_name}_{period}_{today:%Y%m%d}.pdf"` where `safe_name`
    is the child name with non-alphanumeric runs replaced by `_`.
- Registered in `app.create_app()` next to `settings_bp`.

### `templates/dashboard.html` (wire the button)

- In `renderReports()`:
  - Give the child `<select>` an id (`r-child`) and set option values to the child id:
    `<option value="${c.id}">${c.name}</option>`.
  - Add an id to the period select (`r-period` already exists) and the type select
    (`r-type` already exists).
  - Replace the Generate button's `onclick` with `generateReport()`.
- Add `generateReport()`:
  - Read `r-type`. If it is `"Individual Child"`:
    - Read `r-child` value (the child id) and `r-period` value; map the period label
      ("This Week"/"This Month"/"This Term") to `week`/`month`/`term`.
    - If no child selected (`""`) → `alert("Please select a child.")` and return.
    - `window.location = "/api/reports/child/" + id + "?period=" + period;` (triggers the
      browser download via Content-Disposition).
  - Else (Class Summary / Whole School) → keep `showToast('Report generation not yet implemented')`.
- The period `<option>` labels map: This Week→week, This Month→month, This Term→term.

## Dependencies

- Add `reportlab>=4.0` to `requirements.txt`. Pure-Python; no native deps; PyInstaller-friendly.
- Install into the project venv during implementation.

## Testing

**`tests/test_reports.py` (new):**

Pure-function tests (no Flask, no PDF parsing) — the source of truth for content correctness:
1. `test_period_start_windows` — `period_start` returns `today-6/-29/-89` for week/month/term.
2. `test_period_start_invalid_raises` — unknown period raises `ValueError`.
3. `test_build_child_report_aggregates` — with a fixed `child` + a known list of incidents,
   assert `total_incidents`, `per_week_avg`, `top_severity`, `avg_duration`, `top_trigger`,
   `top_type`, `peak_time`, and `incident_rows` ordering.
4. `test_build_child_report_empty` — empty incidents → `total_incidents == 0`,
   `per_week_avg == 0.0`, `top_severity == "—"`, `avg_duration == "N/A"`,
   `pattern_text == "No incidents recorded in this period."`.
5. `test_render_child_report_pdf_returns_pdf_bytes` — `render_child_report_pdf(report)`
   returns `bytes` starting with `b"%PDF"` (smoke test that ReportLab renders).

Endpoint tests (Flask test client):
6. `test_child_report_pdf_download` — `GET /api/reports/child/<id>?period=month` → 200,
   `Content-Type` is `application/pdf`, body starts with `b"%PDF"`,
   `Content-Disposition` contains the sanitized child name and `.pdf`.
7. `test_child_report_default_period` — omitting `period` → 200 (defaults to month).
8. `test_child_report_unknown_child_returns_404`.
9. `test_child_report_invalid_period_returns_400`.
10. `test_child_report_empty_incidents_still_pdf` — a child with no incidents → 200 + `%PDF`.

The `reports.py` pure functions and `Child`/`Incident` fixtures live alongside the existing
`tests/conftest.py` fixtures (`app`, `client`, `child_id`, `room_id`, `staff_id`).

## Done Criteria

- `python -m pytest -q` green (existing 34 + the new report tests).
- From Reports → Individual Child, selecting a child + period and clicking Generate
  downloads a real `.pdf` whose stats/table reflect that child's incidents in the window.
- A child with no incidents in the window still produces a valid PDF (empty-state text).
- Class Summary / Whole School keep the existing "not yet implemented" toast.
- `reportlab` is in `requirements.txt`.
